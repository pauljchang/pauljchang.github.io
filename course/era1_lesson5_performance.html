02

The Accuracy Illusion

In 2016, a well-publicized AI system achieved 99.5% accuracy detecting a rare skin condition affecting 0.4% of the population. Journalists called it superhuman. Dermatologists were impressed. There was one problem: a model that simply said "you don't have it" for every single patient would achieve 99.6% accuracy — slightly better, at zero cost, with zero intelligence.

Accuracy measures the fraction of predictions that are correct. When one class is rare, the "always predict the common class" strategy dominates — not because it's smart, but because it's never wrong about the 99% while ignoring the 1% entirely. This is the accuracy illusion, and it lurks in nearly every real-world ML deployment: fraud detection, medical diagnosis, spam filtering, equipment failure prediction.

The confusion matrix is the cure. Instead of collapsing all outcomes into one number, it separates predictions into four quadrants: things you correctly caught, things you incorrectly flagged, things you missed, and things you correctly ignored. From those four numbers, you can ask more precise questions — and those questions have very different answers depending on what failure means in your application.

"A smoke alarm optimized for accuracy would never ring. It would achieve 99.99% accuracy by correctly predicting 'no fire' every moment of every day. We don't build smoke alarms that way. We shouldn't build classifiers that way either."
📐 The Mathematics — Deriving Metrics from the Four Quadrants ▼ expand

Given a confusion matrix with counts TP, FP, FN, TN:

Predicted + Predicted − Actual + | TP | FN | (all real positives = P = TP+FN) Actual − | FP | TN | (all real negatives = N = FP+TN)

The derived metrics:

Accuracy = (TP + TN) / (TP + FP + FN + TN) ← misleading when imbalanced Precision = TP / (TP + FP) ← "of what I flagged, how much was real?" Recall = TP / (TP + FN) ← "of what was real, how much did I catch?" Specificity = TN / (TN + FP) ← "of all negatives, how many did I correctly ignore?" F1 = 2·TP / (2·TP + FP + FN) ← harmonic mean of precision and recall FPR = FP / (FP + TN) ← false alarm rate (x-axis of ROC curve) TPR = Recall = TP / (TP + FN) ← y-axis of ROC curve

Why harmonic mean for F1? Arithmetic mean (P+R)/2 is fooled by a model with P=1.0 and R=0.001 — it scores 0.5, implying decent performance. Harmonic mean penalizes extreme imbalance: 2·(1.0·0.001)/(1.0+0.001) ≈ 0.002. A model that never misses anything real but only ever makes one prediction is not a good model.

The ROC curve plots TPR (y-axis) against FPR (x-axis) as the decision threshold sweeps from 0 to 1. AUC = Area Under the ROC Curve: 0.5 means the model is no better than random; 1.0 is perfect. A model that scores all positives higher than all negatives has AUC = 1.0 regardless of the threshold chosen.

Lab Exercises

EX 01
📊 The Accuracy Lie
Use Demo 1 to find the exact prevalence where the "always negative" baseline beats a model with 90% sensitivity and 90% specificity.

Set sensitivity to 90% and specificity to 90%. Slowly drag the prevalence slider down from 20%. At what prevalence does the warning appear? Why does the crossover point depend on specificity, not sensitivity?

→ Now set prevalence to 1% and try to build a model (adjust sensitivity/specificity) that beats the baseline in accuracy AND has a recall above 50%. Is it possible? What does the confusion matrix look like?

Key question: A hospital claims their AI model achieves "98.7% accuracy" on a cancer screening test. The cancer affects 1.2% of people screened. Before celebrating, what three questions should you ask about the confusion matrix?

EX 02
🔬 The 512 Audit — Perfect vs. Approximate
Use Demo 2 to find a threshold where the perceptron achieves TP=1, FP=0 for your chosen pattern. Then discover what you give up.

Select the Vertical Bar pattern and run the audit. With default weights, how many false positives are there? Where does the operating point sit on the ROC curve?

→ Raise the threshold until FP=0. The ROC operating point will reach the top-left corner — perfect precision. But did recall stay at 1? (It should — there's only one true positive, and if the weights are correct it should score highest of all 512 inputs.)

→ Now deliberately break the weights: set w₅ (center) to −2.0. Re-run the audit. What happens to the score for the Vertical Bar pattern? Can you still find a threshold that catches it without false positives?

Key question: With only 1 true positive out of 512 inputs, the "always predict negative" classifier has 99.8% accuracy. What is its F1 score? What does that reveal about accuracy?

EX 03
⚖️ The Threshold Decision
Use Demo 3 to find appropriate operating points for three scenarios with very different cost profiles.

Select Cancer Screening. Drag the threshold until recall ≥ 0.95 (you catch 95%+ of cancers). What happens to precision? How many false positives per true positive are you accepting?

→ Switch to Spam Filter. Now drag the threshold until precision ≥ 0.98 (almost no legitimate email is lost to spam). What happens to recall? How much spam gets through?

→ Switch to Criminal Facial Recognition. Where should the operating point be, and why is this scenario categorically different from the medical ones? (Hint: consider who bears the cost of a false positive.)

Key insight to find: The same model — same ROC curve, same AUC — can be the right tool or the wrong tool depending entirely on where you set the threshold and who pays for each type of error.

EX 04
🧮 Build F1 by Hand
Understand why F1 is better than the arithmetic mean of precision and recall.

In Demo 2, tune the weights and threshold so you get a confusion matrix with approximately TP=1, FP=0, FN=0, TN=511. Compute F1 by hand: precision = 1.0, recall = 1.0, F1 = 1.0. ✓

→ Now lower the threshold drastically to get FP ≈ 200. Precision drops to ≈ 1/201 ≈ 0.005, recall stays at 1.0. Compute both the arithmetic mean (P+R)/2 and the harmonic mean F1 = 2PR/(P+R). The arithmetic mean gives ≈ 0.5 — suggesting decent performance. F1 gives ≈ 0.01 — correctly identifying a useless model.

Key question: Why does the harmonic mean penalize extreme imbalances between precision and recall, while the arithmetic mean does not? (Hint: think about what happens to H = 2ab/(a+b) when a→0 but b=1.)

05

Discussion Questions

  • The Legal Standard. "Innocent until proven guilty" is a deliberate policy choice to minimize false positives (wrongful convictions) at the cost of more false negatives (acquitted guilty parties). How does this compare to a medical screening policy? Should the law use different standards for different crimes?
  • COVID-19 Context. During peak transmission, health authorities recommended low-threshold testing (flag broadly, isolate widely). At low prevalence, the same threshold causes enormous disruption for little benefit. What additional information would you need to choose the right threshold for a given moment in an epidemic?
  • The AUC Trap. AUC is often used to compare classifiers. A model with AUC=0.95 sounds better than one with AUC=0.88. But if the second model has much better recall at the specific operating point you need, which is actually better? What does this imply about reporting AUC without specifying the operating point?
  • Algorithmic Accountability. A city deploys facial recognition for policing with a published FPR of 0.1%. There are 1,000,000 innocent people in the city. How many innocent people are falsely flagged per year if the system runs once a day? Who should decide the acceptable FPR — the police department, elected officials, or the public?
  • Connection Forward. In Era 3 you'll study imbalanced learning techniques: oversampling, undersampling, and class-weighted loss functions. Based on what you've learned here, why might you want to weight the loss for false negatives more heavily than false positives during training — and when would that be wrong?
06

Terminology

Confusion Matrix2×2 outcome table
A table that breaks classifier predictions into four categories: TP (correctly flagged positive), FP (incorrectly flagged positive — Type I error), FN (missed positive — Type II error), TN (correctly cleared negative). The confusion matrix contains all information needed to compute every other performance metric.
Accuracy(TP+TN)/N
Fraction of all predictions that are correct. Misleading when classes are imbalanced. If 99% of examples are negative, a model that always predicts "negative" achieves 99% accuracy with zero useful capability.
Accuracy = (TP + TN) / (TP + FP + FN + TN)
PrecisionPositive Predictive Value
"Of everything I flagged as positive, what fraction was actually positive?" Precision punishes false alarms. A spam filter optimized for precision rarely sends real email to spam — but lets lots of spam through.
Precision = TP / (TP + FP)
RecallSensitivity · TPR
"Of everything that was actually positive, what fraction did I catch?" Recall punishes misses. A cancer screening optimized for recall catches nearly all cancers — but also flags many healthy patients for follow-up.
Recall = TP / (TP + FN) = TPR
F1 ScoreHarmonic mean of P and R
A single number combining precision and recall. Uses the harmonic mean so that a model with very high precision but near-zero recall (or vice versa) scores poorly — unlike the arithmetic mean, which can be fooled by extreme imbalances.
F1 = 2 · Precision · Recall / (Precision + Recall) = 2TP / (2TP + FP + FN)
ROC CurveReceiver Operating Characteristic
A curve plotting TPR (recall) on the y-axis against FPR on the x-axis as the decision threshold sweeps from 0 to 1. A diagonal line represents random performance (AUC=0.5). A curve hugging the top-left corner represents near-perfect discrimination (AUC≈1.0). The operating point is the single dot corresponding to your chosen threshold.
AUCArea Under the ROC Curve
A threshold-independent summary of classifier quality. Equals the probability that a randomly chosen positive example scores higher than a randomly chosen negative example. AUC=0.5 → random; AUC=1.0 → perfect. Limitation: a high AUC doesn't tell you whether the model is useful at your specific operating point. Always examine the actual confusion matrix at the threshold you intend to deploy.
