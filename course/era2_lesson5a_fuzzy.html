02

The Imprecision Problem

By the mid-1960s, the symbolic AI program was in full swing. Expert systems encoded knowledge as crisp if-then rules. Logic was Boolean. A patient either had a fever or did not. Blood pressure was either high or normal. Every concept had a sharp boundary, and the boundary was wherever the knowledge engineer drew it.

This bothered Lotfi Zadeh. An Azerbaijani-born Iranian-American mathematician at UC Berkeley, Zadeh worked in control systems — designing the feedback loops that keep aircraft stable and chemical plants safe. His domain was full of concepts that resisted crisp definition. A room was not simply hot or not hot. An aircraft was not simply pitching or not pitching. These were matters of degree, and any control system that pretended otherwise was throwing away information.

In 1965, Zadeh published a paper in the journal Information and Control titled "Fuzzy Sets." Its central proposal was simple: replace the membership function of a classical set — which maps every element to 0 (not a member) or 1 (a member) — with a fuzzy membership function that maps every element to a value anywhere in [0, 1]. A person 180 cm tall might have membership 0.7 in the set "tall people." At 200 cm, membership approaches 1. At 150 cm, it might be 0.1. Tallness is a matter of degree, and the mathematics should say so.

The Western AI and mathematics communities were largely unimpressed. Critics argued that fuzzy logic confused two distinct concepts: vagueness (a concept is inherently graded — "tall" has no sharp boundary) and uncertainty (we don't know which of several crisp states is true — "I'm not sure if it will rain"). Probability theory handles uncertainty; it was not clear fuzzy logic was adding anything. Zadeh's critics, most prominently the statistician William Kruskal and the philosopher Rudolf Carnap, argued the framework was either redundant or incoherent.

"The notion of a fuzzy set provides a convenient point of departure for the construction of a conceptual framework which parallels in many respects the framework used in the case of ordinary sets, but is more general than the latter and, potentially, may prove to have a much wider scope of applicability."

— Lotfi Zadeh, "Fuzzy Sets," Information and Control, 1965

Japan and Taiwan did not wait for the philosophical debate to resolve. Japanese engineers in the 1980s applied fuzzy logic to real control problems with measurable results. The Sendai subway system, which opened in 1987, used a fuzzy logic controller for automatic train operation — smoother acceleration, more precise stopping at platforms, and lower energy consumption than the conventional PID controller it replaced. Hitachi, Panasonic, and Mitsubishi shipped fuzzy controllers in washing machines, air conditioners, and camera autofocus systems. The word ファジー (fuzzy) became a consumer electronics marketing term in Japan. By the early 1990s, Japanese companies were filing thousands of fuzzy logic patents annually while American companies filed dozens.

The Western reversal was partly embarrassment and partly economic pressure. If Japanese washing machines with fuzzy logic were outperforming American ones without it, the philosophical objections began to seem less important. By the mid-1990s, fuzzy logic was mainstream in control engineering worldwide — not because the critics had been refuted, but because the applications worked.

The Mathematics of Fuzzy Logic

Fuzzy logic extends classical Boolean logic in a precise mathematical way. The extension is conservative: when membership values are restricted to exactly 0 or 1, fuzzy logic reduces to classical logic. Everything classical logic can do, fuzzy logic can also do. The reverse is not true.

Fuzzy AND (T-norm)

μ(A∧B) = min(μA, μB)

The AND of two fuzzy values is the minimum. If "hot" = 0.7 and "humid" = 0.4, then "hot AND humid" = 0.4. The weakest condition determines the conjunction.

Fuzzy OR (T-conorm)

μ(A∨B) = max(μA, μB)

The OR of two fuzzy values is the maximum. If "tall" = 0.7 or "heavy" = 0.4, then "tall OR heavy" = 0.7. The strongest condition determines the disjunction.

Fuzzy NOT (complement)

μ(¬A) = 1 − μA

The NOT of a fuzzy value is its complement. If "warm" = 0.6 then "not warm" = 0.4. Note: NOT(NOT(A)) = A — double negation holds, as in classical logic.

These three operations give fuzzy logic its computational backbone. Notice something important: when μA and μB are both restricted to {0, 1}, min() becomes AND, max() becomes OR, and 1−x becomes NOT — exactly classical Boolean logic. Fuzziness is a generalization, not a replacement.

The full fuzzy inference cycle has four stages:

#StageInputOperationOutput
1FuzzificationCrisp sensor value (e.g., 26°C)Apply membership functionsFuzzy memberships (e.g., warm=0.6, hot=0.2)
2Rule EvaluationFuzzy membershipsApply IF-THEN rules using min/maxFiring strength per rule (e.g., rule 1 fires at 0.4)
3AggregationFiring strengths + output fuzzy setsClip each output set, union them allCombined output fuzzy set
4DefuzzificationCombined output fuzzy setCentroid (center of mass)Single crisp output value (e.g., fan speed = 62%)

The last stage — defuzzification — is what makes fuzzy logic practical for real actuators. A fan motor needs a specific voltage, not a fuzzy set. The centroid method computes the center of mass of the aggregated output membership function: it is the value that would balance the shape if it were a physical object resting on a fulcrum. This produces a smooth, proportional output that changes continuously as inputs change — unlike a crisp rule system that snaps between output states at threshold boundaries.

Fuzzy vs. probability — the key distinction. This confused Zadeh's critics for decades. Probability expresses uncertainty about which crisp state is true: "There's a 70% chance this patient has high blood pressure" means we're not sure which of two crisp states (high / not high) is the case. Fuzzy membership expresses gradedness of the concept itself: "This patient's blood pressure is 0.7 high" means the concept "high" is inherently vague and this patient instantiates it to degree 0.7. The first is epistemological — about our knowledge. The second is ontological — about the concept. Both are legitimate. Neither replaces the other.

Lab Exercises

EX 01
〰️ Design Your Own Membership Functions
Using Demo 1 in the lab, classify patient age into pediatric / adult / geriatric.

Open the lab and use Demo 1. Set the domain to age (0–100 years). Position the membership function control points so that: a 5-year-old is fully "pediatric," a 35-year-old is fully "adult," and an 80-year-old is fully "geriatric." Decide where the crossover regions should be.

→ At what age is a patient equally "adult" and "geriatric"? What does that crossover point mean clinically? Is there a meaningful difference between a patient who is 0.5 adult and 0.5 geriatric vs. a patient who is 1.0 adult? What information does the fuzzy representation carry that a crisp threshold discards?

Key question: A hospital protocol says "administer dosage X to adult patients." Under Boolean logic, a 17-year-old gets the child dosage and an 18-year-old gets the adult dosage. Under fuzzy logic, an 18-year-old might be 0.1 adult and 0.9 pediatric. Which system is more clinically honest, and which is more legally convenient? Are these the same thing?

EX 02
⚙️ Trace a Rule by Hand
Work through one full fuzzy inference cycle manually, then verify with Demo 2.

Given: temperature = 26°C, humidity = 75%. Using the membership functions from Demo 2 (you can read off the values from the fuzzification stage), compute by hand:

(a) The membership degree of 26°C in "warm" and "hot"
(b) The membership degree of 75% in "high" humidity
(c) The firing strength of the rule: IF temp IS warm AND humidity IS high THEN fan IS medium
(d) If the "medium" fan output set spans [40, 80] with peak at 60, and fires at strength 0.6, what is the clipped output set?

Key insight: The clipping step is where the firing strength becomes the new maximum of the output set — the output set is "cut" at the firing strength. Multiple rules produce multiple clipped output sets, which are then combined by taking the union (max at each point). Run Demo 2 to verify your hand calculation.

EX 03
⚖️ The Boundary Snap Problem
Use Demo 3 to find the exact input values where Boolean logic snaps and fuzzy logic doesn't.

In Demo 3, set BMI = 25, age = 50, and slowly drag systolic blood pressure from 135 to 145. Observe both panels.

→ At what exact BP value does the Boolean classifier change its output? What is the fuzzy classifier's output at that same value, and how does it compare to 1 mmHg below and 1 mmHg above?

→ Now set BP = 160 (hypertensive) and drag age from 25 to 80. The Boolean classifier treats a 25-year-old and a 79-year-old identically if their BP is the same. The fuzzy classifier does not. What additional clinical information does the fuzzy classifier incorporate, and why does age matter for cardiovascular risk assessment?

Key question: Medical guidelines use crisp thresholds (BP > 140 = hypertensive) for legal and administrative clarity. Fuzzy logic would produce more clinically accurate gradations. What are the arguments for keeping crisp thresholds even when fuzzy ones would be more accurate? This is not a purely technical question.

Discussion Questions

  • Vagueness vs. Uncertainty. Zadeh's critics argued fuzzy logic confuses vagueness with uncertainty. Give your own example of a concept that is vague (inherently graded) and a different example of a situation that is uncertain (we don't know which crisp state is true). Can you find a case where the same real-world situation involves both simultaneously? How would you handle it?
  • The Asian Advantage. Fuzzy logic was invented in the United States, largely ignored by American industry for two decades, and commercialized at scale in Japan and Taiwan. What does this pattern suggest about the relationship between basic research and applied engineering? Can you identify other technologies that followed a similar path — invented in one country, adopted first in another?
  • Connection to Expert Systems. In Lesson 2.1, you saw that expert systems fail at the boundary — MYCIN fires a rule or doesn't fire it, with no gradation. Fuzzy expert systems (rules with fuzzy predicates) can express "IF temperature IS somewhat elevated AND symptoms ARE moderately consistent with bacterial infection THEN diagnosis IS possibly bacterial meningitis (0.6)." What does this gain, and what does it lose, compared to a crisp expert system?
  • Fuzzy Logic vs. Neural Networks. Both fuzzy logic and neural networks can approximate any continuous function. Fuzzy systems are interpretable — you can read the rules. Neural networks are not — you cannot read the weights. For a medical diagnosis system, which property matters more: accuracy or interpretability? Does the answer change depending on whether the system is making a recommendation or making a decision?
  • Connection to Era 3 — Probabilistic Models. In Era 3 you will study Bayesian methods, which handle uncertainty rigorously using probability. Both fuzzy logic and Bayesian inference produce outputs in [0, 1], but they mean different things. Could you combine them — using fuzzy membership functions to define linguistic concepts, and Bayesian inference to update beliefs about which fuzzy category applies? What would such a system look like?

Terminology

Fuzzy SetZadeh, 1965
A set in which membership is a matter of degree rather than binary membership. Defined by a membership function μ: X → [0,1]. Classical sets are the special case where μ maps only to {0,1}. Fuzzy sets allow partial membership: an element can belong to a set to degree 0.7, simultaneously belonging to another set to degree 0.3.
Membership Functionμ: X → [0,1]
The function that defines a fuzzy set by mapping each element of the universe of discourse to its degree of membership. Common shapes: triangular (peak at one value, zero at two others), trapezoidal (flat top between two values), Gaussian (smooth bell curve), sigmoid (smooth step). Shape encodes domain knowledge about how the concept varies.
μ_warm(t) = max(0, min((t−15)/10, (30−t)/5)) % triangular, peak at 23°C
FuzzificationStep 1 of inference
Converting a crisp input value into fuzzy membership degrees by evaluating it against all relevant membership functions. A temperature of 26°C might fuzzify to: cold=0, cool=0, warm=0.6, hot=0.2. One crisp number becomes several partial memberships, one per linguistic term.
DefuzzificationStep 4 of inference
Converting the aggregated fuzzy output set back into a single crisp value suitable for an actuator. The most common method is the centroid (center of mass): integrate x·μ(x) over the output domain and divide by the integral of μ(x). Produces a smooth output that varies continuously as inputs change.
output = ∫ x·μ(x) dx / ∫ μ(x) dx
Linguistic VariableZadeh's key concept
A variable whose values are words or phrases in a natural language rather than numbers. Example: "temperature" takes values {freezing, cold, cool, warm, hot} rather than {15, 20, 25, 30, 35}. Each linguistic value is defined by a fuzzy membership function. Rules are then expressed in natural language: IF temperature IS warm THEN fan IS medium.
T-norm / T-conormFuzzy AND / OR
Generalizations of Boolean AND and OR to fuzzy values. The standard T-norm is min(a,b) — fuzzy AND. The standard T-conorm is max(a,b) — fuzzy OR. Other T-norms exist (product: a·b, Łukasiewicz: max(0, a+b−1)) with different properties. The choice of T-norm affects inference behavior at extreme membership values.
