02

Two Theories of Weakness

After Leo Breiman introduced bagging in 1994, the machine learning community had a powerful tool for reducing variance — making unstable models stable by averaging many of them. But bagging only works on models that have high variance to begin with. It does nothing for a model that is consistently wrong because it is too simple. That is a bias problem, and bias cannot be averaged away.

Yoav Freund and Robert Schapire were working on a different question: can any learning algorithm that does slightly better than random guessing be transformed into an arbitrarily accurate one? Schapire's 1990 boosting-by-filtering paper proved the answer was yes — in polynomial time. The result was surprising enough that many in the community doubted it would translate into a practical algorithm. In 1995, Freund and Schapire published AdaBoost, which did exactly that. It won the Gödel Prize in 2003, given to outstanding papers in theoretical computer science — rare recognition for a machine learning algorithm.

AdaBoost's insight is to train weak learners sequentially, with each learner forced to pay attention to the examples the previous learners got wrong. It does this through example weighting: each training example has a weight, initially uniform. After each round, misclassified examples have their weight increased; correctly classified examples have their weight decreased. The next learner sees a reweighted dataset where hard examples loom large. The final prediction is a weighted vote of all learners, where learners that performed better on their weighted dataset get more say.

"The discovery that any weak learning algorithm can be boosted to achieve arbitrary accuracy is one of the most beautiful results in learning theory. AdaBoost made this constructive."

— Michael Kearns, paraphrased from commentary on the Gödel Prize, 2003

Jerome Friedman's 1999 and 2001 papers reframed boosting as gradient descent in function space. Instead of reweighting examples, gradient boosting fits each new tree to the residuals of the current ensemble — the difference between the true values and what the ensemble currently predicts. For squared error loss, the residuals are literally y − ŷ. For other loss functions, the "residuals" are generalized negative gradients. Each tree is a small step in the direction that reduces the ensemble's total loss. A learning rate (shrinkage parameter) controls how large each step is: small learning rate means slower learning but better generalization.

XGBoost (2016) added regularization terms to the objective function, approximate split-finding algorithms, and better handling of missing values and parallelism. The result dominated Kaggle competitions for years — not because of a new theoretical idea, but because every practical detail was engineered carefully. "When in doubt, use XGBoost" became a running joke in competitive machine learning that contained a lot of truth.

Bagging

  • Parallel — models are independent
  • Reduces variance, bias unchanged
  • Works best on high-variance models
  • Cannot overfit by adding more models
  • All models weighted equally
  • Robust — errors in any one model don't propagate

Boosting

  • Sequential — each model depends on all previous
  • Reduces bias, some variance reduction too
  • Works best on high-bias models (stumps)
  • Can overfit with too many rounds
  • Models weighted by performance
  • Fragile — early errors can compound

Lab Exercises

EX 01
🎲 What Bagging Stabilises
Use Demo 1 to observe variance reduction in action — and identify what bagging cannot fix.

In Demo 1, draw the first 3 bootstrap samples one at a time. Note how much the individual model predictions (thin coloured lines) vary from each other. Now add 7 more. Does the ensemble average (thick white line) move much between samples 8, 9, and 10?

→ The individual models have high variance — each bootstrap sample produces a noticeably different fit. The ensemble average has much lower variance — it barely moves as you add more samples. This is variance reduction in action.

Key question: If all five individual models are systematically biased in the same direction — say, all of them predict values that are consistently too high on the right side of the chart — will the ensemble average correct this? Why or why not? What does this tell you about when bagging helps and when it doesn't?

EX 02
🔁 Tracing AdaBoost
Use Demo 2 to follow the weight update logic and understand why harder examples dominate later rounds.

In Demo 2, step through all 5 rounds. After round 1, identify which examples were misclassified (they grow larger). After round 2, do the same examples get misclassified again, or different ones? Why does the algorithm sometimes struggle with the same examples across multiple rounds?

→ Note the α (voting weight) for each round. A round with a very high α means the stump was very accurate on the weighted dataset. A round with a low α means the stump barely outperformed random guessing on that weighting. Does α generally increase or decrease over rounds? Why?

Key question: AdaBoost assigns exponentially increasing weight to misclassified examples. What happens if a single example is mislabelled in the training data (a genuine error, not a hard case)? How would this affect the algorithm's behaviour — and what does this reveal about boosting's sensitivity to noisy labels compared to bagging?

EX 03
⚖️ The Overfitting Asymmetry
Use Demo 4 to find the round at which boosting starts to overfit — and confirm that bagging never does.

In Demo 4, drag the slider from left to right. At what number of rounds does the boosting boundary start memorising individual training points rather than capturing the general pattern? What happens to the test accuracy at that point?

→ Now look at the bagging panel at the same slider position. Does the boundary show the same tightening? Does test accuracy fall?

Key insight: Bagging cannot overfit by adding more trees — the ensemble average of a thousand trees is no more overfit than the average of ten. Boosting can overfit because each subsequent tree is deliberately targeting the residuals of an already-overfit ensemble, compounding its errors. The learning rate (shrinkage) is the primary tool for controlling this. A smaller learning rate means each tree contributes less — you need more trees, but you overfit later.

05

Discussion Questions

  • The Theoretical Surprise. Schapire's 1990 result — that any weak learner can be boosted to arbitrary accuracy — seemed too good to be true when it was published. The proof assumes unlimited data and a weak learner that can always do slightly better than random on any weighted distribution. In practice, neither assumption holds perfectly. Why does AdaBoost still work so well empirically despite these idealised assumptions?
  • Noise Sensitivity. Bagging is robust to label noise — a mislabelled example might appear in some bootstrap samples but not others, diluting its effect. Boosting is sensitive to label noise — a mislabelled example gets higher and higher weight as the algorithm keeps failing to classify it correctly. What does this suggest about the right choice of method for a medical dataset where some diagnoses in the training data may be incorrect?
  • XGBoost and the Engineering Lesson. XGBoost didn't introduce a fundamentally new algorithm — it took gradient boosting and engineered it carefully. Regularisation terms, approximate split finding, column subsampling, handling of missing values. The result was qualitatively better in practice. What does this suggest about the relative importance of algorithmic novelty vs. engineering quality in applied machine learning?
  • Connection to Neural Networks. Gradient boosting fits residuals iteratively, making each model a small correction to the previous ensemble. Dropout in neural networks randomly drops units during training, producing an ensemble effect similar to bagging. Do you see the same bias/variance distinction in these two regularisation techniques? Can you describe dropout as "neural bagging"?
  • Tabular Data's Persistence. On structured tabular data, gradient boosted trees (XGBoost, LightGBM, CatBoost) consistently outperform deep neural networks, even with large datasets. On images, text, and audio, deep networks dominate. What property of tabular data might explain why boosted trees remain competitive — and what property of image/text data makes deep networks so dominant there?
06

Terminology

AdaBoostFreund & Schapire, 1995
The first practical boosting algorithm. Trains weak classifiers sequentially, reweighting examples after each round so misclassified examples receive higher weight. Final prediction is a weighted majority vote where learner weight α = ½ log((1−err)/err). A learner with 0% error gets infinite weight; one with 50% error (random) gets zero weight.
α_t = ½ · log((1 − err_t) / err_t) w_i ← w_i · exp(−α_t · y_i · h_t(x_i))
Gradient BoostingFriedman, 1999–2001
Reframes boosting as gradient descent in function space. Each new tree fits the negative gradient of the loss function — the direction that reduces total loss most. For squared error loss this is exactly the residuals y − ŷ. A learning rate η scales each tree's contribution, trading speed for generalization.
F_m(x) = F_{m-1}(x) + η · h_m(x) where h_m fits the residuals: r_i = y_i − F_{m-1}(x_i)
Weak LearnerBetter than random
A classifier or regressor that performs only slightly better than chance — for example, a decision stump (single-split tree) that achieves 55% accuracy on a binary classification task. Boosting's theoretical guarantee: any weak learner (better than 50%) can be combined into an ensemble with arbitrary accuracy, given enough data and rounds.
Learning RateShrinkage — boosting's regulariser
A scalar η ∈ (0,1] that scales each tree's contribution in gradient boosting. Small η means each step is small — requires more trees but generalises better (less overfitting). There is a strong empirical trade-off: lower learning rate + more trees ≈ higher learning rate + fewer trees in training accuracy, but the low-rate version almost always generalises better. Typical values: 0.01–0.3.
ResidualsWhat the ensemble still gets wrong
In gradient boosting, the vector of differences between true labels and the current ensemble's predictions: r_i = y_i − F(x_i). Each new tree fits these residuals, approximating the remaining error. As rounds increase, residuals shrink — the ensemble progressively explains more of the structure in the data. If residuals stop shrinking, the ensemble has converged or begun overfitting.
XGBoosteXtreme Gradient Boosting, 2016
An engineered implementation of gradient boosting that adds: L1/L2 regularization on leaf weights, approximate split-finding for large datasets, column subsampling (like Random Forests), efficient handling of missing values, and parallelisation. The engineering improvements — not algorithmic novelty — account for its dominance on tabular data. Won more Kaggle competitions than any other single algorithm.
